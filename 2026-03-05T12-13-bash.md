### Условный оператор
```
if команда1
then
команды
elif команда2
then
команды
else
команды
fi
```
Пример 1:
```bash
if pwd
then
echo "it works"
fi
```
Пример 2:
```bash
user=kovalev_m
if grep $user /etc/passwd;
    then
      echo "user $user exists";
    else
      echo "user $user does not exist";
fi
```
Пример 3:
```bash
#!/bin/bash
user=kovalev_m
if grep $user /etc/passwd; then
  echo "user $user exists"
elif ls /home; then
  echo "user $user does not exist but his directory does"
else
  echo "user $user does not exist"
fi
```

### Сравнение чисел
 - `n1 -eq n2` истинно, если `n1` равно `n2`
 - `n1 -ge n2` истинно, если `n1` больше или равно `n2`
 - `n1 -gt n2` истинно, если `n1` строго больше `n2`
 - `n1 -le n2` истинно, если `n1` меньше или равно `n2`
 - `n1 -lt n2` истинно, если `n1` строго меньше `n2`
 - `n1 -ne n2` истинно, если `n1` не равно `n2`

Пример
```bash
val1=5
if [ $val1 -gt 5 ]; then
    echo "$val1 is greater than 5"
else
    echo "$val1 is not greater than 5"
fi
```

### Сравнение строк
 - `str1 = str2` истинно, если строки идентичны
 - `str1 != str2` истинно, если строки НЕ идентичны
 - `str1 \< str2` истинно, если `str1` в лексикографическом порядке должна быть раньше, чем `str2`
 - `str1 \> str2` истинно, если `str1` в лексикографическом порядке должна быть после `str2`
 - `-n str1` истинно, если `str1` не пустая
 - `-z str1` истинно, если `str1` пустая

В лексикографическом порядке строчные буквы располагаются раньше заглавных.

При сравнениях строк их лучше оборачивать в кавычки

Пример (не работает, если `val2` содержит пробелы)
```bash
val1=foo
val2="foo bar"
if [ $val1 \> $val2 ]
then
    echo "$val1 is greater then $val2"
else
    echo "$val2 is greater then $val1"
fi
```
Выводит
```
bash: [: too many arguments
foo bar is greater then foo
```

Исправленный пример:
```bash
val1=foo
val2="foo bar"
if [ $val1 \> "$val2" ]
then
    echo "$val1 is greater then $val2"
else
    echo "$val2 is greater then $val1"
fi
```
Работает как надо
```
foo bar is greater then foo
```

### Проверки файлов
 - `-d file` истинно, если `file` существует и является папкой
 - `-e file` истинно, если `file` существует (либо файл, либо папка)
 - `-f file` истинно, если `file` существует и является файлом
 - `-r file` истинно, если `file` существует и доступен для чтения (либо файл, либо папка)
 - `-s file` истинно, если `file` существует и не пуст
 - `-w file` истинно, если `file` существует и доступен для записи
 - `-x file` истинно, если `file` существует и является исполняемым
 - `file1 -nt file2` истинно, если `file1` новее, чем `file2`
 - `file1 -ot file2` истинно, если `file1` старше, чем `file2`
 - `-O file` истинно, если файл существует и его владелец -- текущий пользователь
 - `-G file` истинно, если файл слуществует и его группа соответствует группе текущего пользователя
